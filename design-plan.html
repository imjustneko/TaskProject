Taskyy · Design Plan

Calm tasks, together.

A UI/UX plan for a social productivity app that sits between Notion, Discord, and Todoist — designed to feel quiet, focused, and friendly rather than performative.

01 · Concept & positioning

Taskyy is a social productivity app: a to-do list that knows your friends. The thesis: most productivity apps are lonely, and most social apps make you perform. Taskyy puts low-stakes shared activity — reading the same book, running the same program, writing 25 minutes before work — at the center of getting things done.

The visual mood is borrowed from Things and Linear (calm, restrained, almost stationery-like), the structure from Notion (sidebar, rooms, fluid content), the social shapes from Discord (rooms with chat, presence, member rosters), and the warmth from Instagram / BeReal (status emoji, the optional photo-on-task "daily memory").

Product principles

  • Small actions, visible together. Checking off a task shouldn't feel lonely.
  • Quiet by default. No streaks, no badges, no nags. Presence is opt-in.
  • One screen at a time. The dashboard tells you what's next, not everything.
  • Photos as memory, not metric. Daily-memory photos are personal first, social second.

02 · Audience & jobs-to-be-done

Primary user

25–40 yo professionals & students who already use a to-do app casually and have a small friend group they'd happily nudge.

Job to be done

"Help me do the same small good things every day — and let me feel like I'm not doing them alone."

Top-of-mind jobs

  1. Plan my day — quickly add tasks for today / future / someday.
  2. See what my friends are up to right now (without messaging them).
  3. Do an activity together on a schedule (read, run, study, write).
  4. Look back at a small visual diary of what I did this week.

03 · Main user flow

First-run flow

Landing→ Sign up→ Pick handle & status→ Add 3 friends (search/handle)→ Suggested rooms→ Empty dashboard

Daily loop

Open · see today→ Glance friends & rooms→ Check off task→ (optional) snap a memory→ Chat in room

Room flow

Friend profile→ Invite to room / accept→ Room view: checklist + chat→ Tick own tasks→ See per-member progress

04 · Page-by-page layout

Shell (everywhere)

  • Left sidebar (240px) — Brand · New task CTA · Personal nav (Home/Today/Plans/Friends/Profile) · Rooms list · Me + status footer.
  • Top bar (52px) — Page title + breadcrumb · Status pill · Search · Notifications · Theme toggle.
  • Content area — Page-specific. Scrolls independently.

Landing & Auth

  • Landing — Split hero: brand panel with sample room chips on the left, value prop & CTA on the right.
  • Login / Register — Single form, mode toggle inside it. Magic-link & SSO secondary.

Home dashboard

2-column inside content. Left: Today's tasks (top 5) → Upcoming (next 4). Right: Friends · now → Your rooms with own/room progress → Friend activity feed. Top strip is a single day-progress bar.

Today

Narrow column (max ~840px). Eyebrow + date. Two sections: To do and Completed. Inline quick-add via N shortcut.

Plans

Narrow column. Tasks grouped by Tomorrow / Friday / Saturday / Next week / Someday. Drag-to-reschedule (v1).

Friends

Search bar + All / Online tabs. Requests card at top with Accept/Decline. Grid of friend cards below — avatar, bio, status pill, quick message.

Friend profile

Hero with banner (avatar-derived gradient), name, mutuals, bio, current status. Two columns below: Shared rooms, Public tasks. Message + Invite-to-room actions in the hero.

Shared Room

Two-column inside the content: left for tasks & progress, right (~320px) for member list and chat. Hero header: emoji, name, description, active-now pill, members. Below: Your progress strip → Tasks (each row shows per-member avatar dots) → Member progress leaderboard. Chat is Discord-style: avatar + grouped messages + composer with attach/image/send.

Profile (edit)

Personal info card (photo, display name, handle, bio) + Privacy card (status visible, share shared tasks, daily memory photos public, online indicator). Update-status entry point.

Modals

  • Create task — Title, notes, when (Today / Tomorrow / etc.), time, category, priority, share toggle, daily memory toggle.
  • Update status — 8 preset chips + custom emoji + freeform label.
  • Invite to room — Search + multi-select friends.

Admin (post-MVP)

  • Admin dashboard — Stat cards (Total / Active / Tasks / Completed / Rooms) + 7-day chart + recent signups.
  • Admin users — Filterable table (search, status, actions: view / block / delete).
  • Reports queue — Inappropriate content review with image preview & resolution actions.

06 · Data model

Relational core. Cosmetic flags (privacy, presence) live on User. Rooms have many-to-many members and own their checklists & chat. Tasks belong to one user, optionally referencing a room.

Type

  • Geist Sans — UI body & headings. Sizes: 11, 12, 13.5 (body), 14, 16, 22, 26.
  • Geist Mono — Times, counts, codes, keyboard shortcuts.
  • Letter-spacing tightens by –0.01em to –0.025em as size grows.

Spacing & radius

4px spacing scale: 4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 48. Radius scale: 4 · 6 · 8 · 10 · 14 · 18 · 999. Cards default to 14px, room/profile hero to 18px, modals to 18px.

Density

Three densities (compact / regular / comfy) switch the row height and gutter sizes via data-density on :root. Defaults: row 38px, gap 14px.

Motion

One easing curve: cubic-bezier(.2, .8, .2, 1). Two durations: 120ms (interactions, hover) and 200ms (entry / route changes). No spring physics, no parallax.

09 · Feature priorities

MVP — first month

MVP Ship the daily loop end-to-end so one person and one friend can use it together.

  • Email + password auth · password reset.
  • Profile (name, handle, bio, avatar upload) + Status (presets + custom).
  • Tasks: create, edit, complete, delete, today/plans buckets, basic categories, priority.
  • Friend search by handle, request / accept / decline.
  • Friend profile (read-only); current status visible.
  • Rooms: create, invite, leave; shared checklist with per-member completion; basic room chat (text only).
  • Dashboard (today + upcoming + friends now + your rooms).
  • Light/dark themes.

v1.0 — month 2

v1 Make it pleasant to live in.

  • Daily memory photos on tasks (upload + compression + private/share toggle).
  • Friend activity feed on dashboard (shared task completion, memories).
  • Direct messages (1:1 chat).
  • Push / email reminders for tasks & room nudges.
  • Recurring tasks (daily / weekly / weekdays).
  • Search across tasks & messages.
  • Calendar view of plans.

v1.5 — month 3

v1.5 Lean into "together".

  • Image attachments in chat.
  • Room templates (Couch-to-5K, Morning Pages, book clubs).
  • Public / discoverable rooms (opt-in).
  • Weekly digest email (your week in tasks & memories).
  • Mobile-responsive web app (then native shells).

Admin track — alongside v1

Admin Required before opening signup.

  • Admin dashboard with stat cards + last 30 signups.
  • Users table (search, filter, view, block, delete).
  • Content reports queue (text + image, dismiss / remove / suspend).
  • Audit log of admin actions.

10 · Frontend handoff checklist

  1. Lift the design tokens from styles.css in this prototype into your Tailwind/CSS-vars config.
  2. Port the atoms — Button, Avatar, Badge, Checkbox, Toggle, Icon, Input, Modal — first; nothing else can be built without them.
  3. Build the app shell (sidebar + topbar) with route stubs; wire empty page components in.
  4. Backend before social: Tasks CRUD & Today/Plans views work without realtime.
  5. Layer in friends & profile (read-only) — still no realtime.
  6. Add presence (status + online indicator) — first realtime touchpoint.
  7. Build the Room view last and most carefully — it carries the product. Realtime tasks, chat, member progress.
  8. Photos & memories after rooms — they need storage + moderation infra.
  9. Admin pages in parallel, gated by role.
